# Translation and i18n

Translation and i18n support applies from `v2.7.0`.

## Config

Set the default language and available languages in config:

```json
"site": {
  "language": "en",
  "languages": ["en", "no"]
}
```

When more than one language is configured, the header shows a language selector.

## Content Fields

Use `lang` to mark an entry's language:

```md
---
title: About
slug: about
lang: en
translationKey: about
---
```

Translated variants should share `translationKey`:

```md
---
title: Om
slug: om
lang: no
translationKey: about
---
```

Entries without `lang` are shown for every language. This is useful for language-neutral downloads or assets.

## Filtering

The active language filters:

- module lists
- module detail pages
- collection lists
- collection detail pages
- search results

The active language is stored in browser local storage.

## Static Export

Static export includes all published languages in `site.json` and `search-index.json`. The browser applies the active language filter.

## Analytics

`dist/data/analytics.json` includes language counts generated from content metadata. No tracking scripts are added.

## v3 Preparation Notes

- Set `site.languages` before adding translated content.
- Use explicit `lang` for translated entries. Entries without `lang` are language-neutral and show in every language.
- Use the same `translationKey` for translated variants of the same entry.
- Prefer per-language slugs when the visible URL should be localized.
